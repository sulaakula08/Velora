# Velora 🤝

Персональный AI-ассистент, который живёт в **Telegram** и одинаково хорошо понимает **русский и казахский**. Аналог [getfolk.app](https://getfolk.app), но канал общения — Telegram, а целевой рынок — Казахстан.

Velora умеет:

- 💬 **Чат-ассистент** — обычный диалог с памятью на каждого пользователя, ответы на языке последнего сообщения (RU/KK определяется автоматически).
- 📌 **Напоминания** — «напомни завтра в 18:00 позвонить маме». Планировщик раз в минуту проверяет БД и присылает напоминание в тот же чат (часовой пояс `Asia/Almaty`).
- 📝 **Заметки** — «запомни, что переговоры перенесли на пятницу».
- 👥 **Контакты / мини-CRM** — «сохрани: Ержан — директор завода» и «что я знаю про Ержана?».
- 🧾 **Recap** — перешли переписку или опиши встречу, бот выделит договорённости и следующие шаги и сохранит их к контакту.
- ✍️ **Черновики сообщений** — «напиши вежливый отказ клиенту на казахском».
- 🔎 **Веб-поиск** — актуальные данные через Gemini Google Search: погода, курсы валют, новости, цены, часы работы, места рядом.
- 🖼️ **Мультимодальность** — понимает присланные фото, голосовые, аудио и документы (PDF/текст): распознать чек, разобрать скрин переписки, кратко изложить документ.
- ✅ **Задачи + follow-up** — дела со статусом «сделал/не сделал» и сроком; бот сам переспрашивает о невыполненном.
- 🌅 **Проактивный брифинг** — утром сам пишет: задачи и напоминания на день (`/briefing on|off|<час>`).
- 🧠 **Память о тебе** — помнит имя, город, предпочтения и цели, учитывает во всех ответах.
- ⚡ **Быстрый захват** — форвардни ссылку/сообщение/фото, бот сам разложит в задачу/заметку/контакт/напоминание.
- 📨 **Отправить человеку через Velora** — «напиши @aigul: …»: если получатель тоже пользуется ботом, сообщение доставляется ему с подписью от тебя; если нет — бот даёт готовый текст для ручной пересылки (первый кирпич мультиплеера).
- 📅 **Google Календарь** и 📧 **Gmail** — после `/connect_google`: создать/посмотреть события, сводка входящих, отправка писем (OAuth, токены на пользователя).
- 🌐 **Многоязычный интерфейс** — все служебные строки в `locales/ru.json` и `locales/kk.json`, ручное переключение командой `/language`.

## Технологии

- **Backend:** Node.js + TypeScript
- **Telegram:** `node-telegram-bot-api` (long polling — без webhook, проще запускать локально)
- **LLM:** Google Gemini API (модель `gemini-2.5-flash` по умолчанию) с function calling
- **БД:** SQLite через `better-sqlite3`
- **Планировщик:** `node-cron`
- **Логи:** `pino`

## Шаг 1. Получить токен Telegram-бота у @BotFather

1. Открой в Telegram чат с [@BotFather](https://t.me/BotFather).
2. Отправь команду `/newbot`.
3. Введи **имя** бота (любое, например `Velora Assistant`).
4. Введи **username** — он должен заканчиваться на `bot` (например `my_velora_bot`).
5. BotFather пришлёт строку вида `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxx` — это `TELEGRAM_BOT_TOKEN`.

## Шаг 2. Получить ключ Google Gemini API

1. Зайди на <https://aistudio.google.com/apikey> (нужен аккаунт Google).
2. Нажми **Create API key**.
3. Скопируй ключ — это `GEMINI_API_KEY`.
4. У Gemini есть бесплатный тариф, отдельная карта для старта не требуется.

## Шаг 3. Запуск локально

Требуется **Node.js 18+** (проверено на Node 22).

```bash
# 1. Установить зависимости
npm install

# 2. Создать .env из шаблона и вписать свои ключи
cp .env.example .env
#   (в Windows PowerShell: Copy-Item .env.example .env)
#   затем открой .env и заполни TELEGRAM_BOT_TOKEN и GEMINI_API_KEY

# 3. Запустить в режиме разработки (автоперезапуск при изменениях)
npm run dev
```

В логах должно появиться:

```
Velora готов к работе. Напиши боту в Telegram.
```

Теперь открой своего бота в Telegram и напиши `/start`.

### Продакшн-сборка

```bash
npm run build   # компиляция TypeScript → dist/
npm start       # запуск собранной версии
```

## Как это устроено (кратко для ревью)

- **Определение языка ответа** делает сам Gemini (инструкция в системном промпте — отвечать на языке последнего сообщения). Отдельного классификатора нет.
- **Язык интерфейса** (служебные строки) определяется простой эвристикой по казахским буквам (`src/i18n/i18n.ts`) и хранится на пользователя в БД; `/language` фиксирует его вручную.
- **Инструменты** реализованы как отдельные модули с единым провайдер-независимым интерфейсом `Tool` (`name`, `description`, `input_schema`, `execute`) и собраны в `src/tools/registry.ts`; там же JSON-схема конвертируется в формат function declarations Gemini. Ядро (`src/ai/agent.ts`) с инструментами работает единообразно, поэтому добавить Google Calendar/Gmail через OAuth в будущем можно, не переписывая цикл диалога.
- **Напоминания** хранятся с абсолютным временем (epoch ms); `node-cron` раз в минуту выбирает наступившие и отправляет. Абсолютное время из фраз вроде «завтра в 18:00» вычисляет Gemini, зная текущее время `Asia/Almaty` из системного промпта.

## Вне скоупа этого MVP (осознанно)

- Браузерная автоматизация (бронирование столиков и т.п.).
- Crew-механика между несколькими пользователями.
- OAuth-интеграции с Google — заложена только архитектура под инструменты (см. `src/tools/`).
